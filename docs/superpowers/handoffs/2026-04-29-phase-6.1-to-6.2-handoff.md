## Visual / live-API smoke checklist (PENDING — for the human)

These are plan steps 5.2 and 5.3. Run before tagging.

### 5.2 — End-to-end happy path

Prerequisites: OpenRouter key with $0.20+ balance, Linear key with team access,
1-2 minute Russian-language audio file.

1. Fresh app launch. Bottom row reads `[Сохранить] [Копировать] [Извлечь задачи] [История] [Audio Cutter]`.
2. "Извлечь задачи" is disabled (greyed).
3. Pick an audio file → transcribe (any path: local or cloud).
4. After transcription completes, "Извлечь задачи" enables.
5. Click it. Dialog opens. Header: `Модель: [...] Команда: [...] [↻] [Извлечь]`.
6. Cost hint shows "Стоимость ≈ $X.XX".
7. Team dropdown populates within ~5s (or shows "(нет команд)" if user has no Linear teams).
8. Pick a team, click Извлечь.
9. Status badge cycles: Запрос к Linear → Запрос к OpenRouter → ✓ Извлечено N задач.
10. JSON textbox populates with valid JSON containing N tasks.
11. Footer reads `Сохранено: history/<...>/tasks_raw.json`.
12. Open the file on disk — content matches what's in the textbox.
13. Open `config.json` — `tasks_recent_models` reflects the model used (if it was custom-typed).

Acceptance: at least 1 task in `tasks_raw.json` with title in Russian.

### 5.3 — Edge cases

For each, the app should surface a clean error (red status, no traceback, no freeze):

- a) Wrong OpenRouter key (mangle in Settings, save, retry).
- b) Wrong Linear key (mangle, retry).
- c) No internet (kill Wi-Fi mid-extract).
- d) Custom model slug that doesn't exist (`anthropic/totally-fake-model`).
- e) Empty transcript (paste empty text, click Извлечь задачи) — gate-check fires.
- f) Cancel-on-close: click Извлечь on a long transcript, then [Закрыть] mid-extract. Dialog closes immediately. Check `logs/app.log` — no traceback. App stays responsive.

## Carry-forward recommendations from Phase 6.1 reviews

These were flagged as **Minor** during code-quality reviews and not blocking
for ship. Fold into Phase 6.2 work where natural.

### From Task 0 (carry-forwards cleanup) review

1. **`OpenRouterClient.complete()` — `is not None` vs `or` for `error.message`** (`tasks/openrouter_client.py` line ~171). Currently `detail = err or "..."` falls back to "unexpected response shape" if `err` is empty string. Real OpenRouter messages are always non-empty so this is theoretical, but a tighter `if err is not None` check would be more precise.
2. **No `__cause__` assertion in OpenRouter tests** (`tests/test_tasks_openrouter_client.py`). New tests check the raised message but don't verify the original exception is chained. Consistent with existing tests but worth considering as a convention update.

### From Task 1 (persistence) review

3. **`_RAW_FIELDS` ↔ `Task.to_dict()` coupling has no comment** (`tasks/persistence.py:35`). The comprehension `{k: full[k] for k in _RAW_FIELDS}` will raise `KeyError` at runtime if a field is removed from `to_dict()` without updating `_RAW_FIELDS`. This is the safer failure mode (loud crash) but the implicit coupling is undocumented.
4. **`data.pop("tasks", [])` mildly surprising idiom** (`tasks/persistence.py:83`). `pop` mutates the local dict; harmless but `{k: v for k, v in data.items() if k != "tasks"}` would be clearer.
5. **Redundant manual `monkeypatch.setattr` restore** (`tests/test_tasks_persistence.py:89`). monkeypatch teardown is automatic; the manual restore is dead code.
6. **`load_tasks_raw` `OSError` labeled "malformed"** (`tasks/persistence.py:81`). A permission error after `is_file()` would be reported as "malformed" — slightly misleading. Split into separate `except` clauses with distinct messages if the distinction ever matters.

### From Task 2 (extractor) review

7. **Stale inline comment "we detect via '400' in the error message"** (`tasks/extractor.py` line ~260). After the fix, detection is `"вернул 400:"`, not just `"400"`. Update opportunistically.

### From Task 3 (app wiring) review

8. **Gate-check ordering should be commented** (`ui/app.py` `_open_extract_tasks_dialog`). Order is API keys → transcript → folder. Current ordering matches expected user friction; documenting it prevents arbitrary reordering.
9. **Warning title "Нет папки истории" could be more precise** — when the user loads a transcript without an audio file, "истории" suggests history is missing rather than a save target. Consider "Нет сохранённой записи".
10. **`=` alignment in API-key block** (`ui/app.py` `_open_extract_tasks_dialog`). The `openrouter_key = ...` / `linear_key     = ...` vertical alignment isn't the project's style. Plan-prescribed formatting; cosmetic.

### From Task 4 (dialog) review

11. **`option_menu` imported but unused** (`ui/dialogs/extract_tasks.py:34`). Plan-prescribed (kept for Phase 6.2 dropdown). Add `# noqa: F401  # reserved for Phase 6.2 dropdown` to silence linter and document intent.
12. **`import logging` inside except clause** (`ui/dialogs/extract_tasks.py:350`). Should be at module top for consistency with other files. The bootstrap worker's broad `except Exception` doesn't log either; symmetry would be nice.
13. **`_active_clients` race fragility**. The `if c in self._active_clients: self._active_clients.remove(c)` check-then-remove is not atomic. Today's code only ever has unique objects, but a future change introducing concurrent workers with overlapping clients could trigger `ValueError`. A `threading.Lock` around mutations would harden this cleanly.
